# 메타 레이밴 디스플레이 도슨트 테스트 앱 — 구현 요청

## 0. 배경 / 목적
박물관 도슨트 시스템 프로토타입의 1차 테스트용 웹앱입니다.
메타 레이밴 디스플레이 글래스에서 웹앱 URL로 접속해 사용하며, BLE 비콘 8개를 작품 위치에 배치해 두고
관람객(글래스 착용자)이 특정 비콘 근처에 가면 해당 작품에 대한 안내 텍스트가 자동으로 화면에 표시되도록 합니다.

## 1. 핵심 동작 플로우
1. 페이지 진입 시 "BLE 스캔 시작" 버튼 1개만 보이는 대기 화면 표시
   (Web Bluetooth는 사용자 제스처로만 스캔을 시작할 수 있으므로)
2. 버튼 클릭 시 BLE 스캔 시작, 화면은 "스캔 중..." 표시로 전환
3. 이름이 "01" ~ "08" 인 BLE 디바이스 중 신호가 가장 강한(RSSI 가장 높은) 디바이스를 "현재 위치"로 판단
4. 현재 위치가 바뀌면(예: 03 → 05) 화면 내용을 새 비콘 내용으로 교체하고 스크롤을 처음부터 다시 시작
5. 화면 표시 내용
   - 스크롤 시작 부분: "0n번 BLE에 왔습니다"
   - 그 아래로 "0n테스트용 스크립트" 텍스트가 화면을 여러 번 채울 만큼 충분히 길게 이어짐
   - 전체 텍스트가 아래에서 위로 천천히 흐르는 텔레프롬프터/엔딩크레딧 방식으로 자동 스크롤
6. 동시에 음성으로도 동일 텍스트를 읽어줌 (TTS). 안되면 비콘 진입 시 테스트용 비프음만 재생하는 옵션도 같이 구현

## 2. 화면 디자인 규칙
- 배경: 완전 검정 (#000000)
- 텍스트: 흰색, 충분히 큰 폰트
- 글래스 디스플레이 해상도는 600x600px 정사각형으로 가정하고, 그 비율의 뷰포트에 맞춰 디자인
  (가로 스크롤 없이 세로로만 흐르게)
- 헤더 텍스트("0n번 BLE에 왔습니다")는 본문보다 조금 더 크고 굵게
- 본문 반복 텍스트는 한 줄에 욱여넣지 말고 줄바꿈해서 세로로 보기 좋게 나열
- 가장자리 40~50px 정도는 안전영역으로 비워두어 글래스에서 잘리지 않게 할 것

## 3. BLE 스캔 구현 메모 (중요)
- `navigator.bluetooth.requestDevice()`는 1회성 페어링용이라 자동/지속 스캔에 부적합함
- 대신 `navigator.bluetooth.requestLEScan()` (실험적 API)를 사용해서 advertisement 패킷을 지속적으로 watch
- 이 API는 현재 Chrome에서 `chrome://flags`의 "Experimental Web Platform features"를 켜야 동작함
  → 코드에 반영할 필요는 없지만, README에 "테스트 기기에서 이 플래그를 켜야 한다"는 안내를 적어둘 것
- advertisement 이벤트에서 `device.name`과 `rssi` 값을 받아서 8개 후보 중 가장 강한 신호를 선택
- 일정 시간(예: 2~3초) 동안 동일 비콘이 가장 강한 신호로 유지될 때만 "현재 위치"로 확정
  (노이즈로 인한 깜빡임 방지)

## 4. 테스트/디버그 모드 (필수)
- 실제 BLE 비콘 8개가 항상 준비되어 있지 않을 수 있으므로, 키보드 숫자 1~8을 누르면 해당 번호 비콘에
  진입한 것처럼 시뮬레이션하는 기능을 반드시 추가
- 화면 한쪽에 작게 "디버그 모드 ON/OFF" 토글, 현재 감지된 디바이스 목록과 RSSI 값을 보여주는
  디버그 패널도 추가 (실사용 시엔 숨겨도 됨)

## 5. 사운드 / TTS
- 기본: Web Speech API의 `SpeechSynthesis`로 한국어(ko-KR) voice를 사용해 헤더+본문 텍스트를 순서대로 읽음
- 설정에서 TTS On/Off 토글 가능하게
- TTS Off 시: 비콘 진입 시점에 짧은 비프음(Web Audio API로 생성한 간단한 톤) 1회 재생
- 스크롤 속도와 TTS 속도를 엄격히 동기화할 필요는 없음 (1차 테스트이므로 대략 비슷한 속도면 충분)

## 6. 데이터 구조 예시
```js
const beacons = [
  { id: "01", title: "01번 BLE에 왔습니다", bodyLine: "01테스트용 스크립트" },
  { id: "02", title: "02번 BLE에 왔습니다", bodyLine: "02테스트용 스크립트" },
  // ... 08까지 동일 패턴
];
```
- `bodyLine`을 화면에 넣을 땐 코드에서 N회(예: 40~60회, 줄바꿈 포함) 반복해서
  충분히 긴 스크롤 콘텐츠를 자동 생성할 것. 긴 문자열을 직접 하드코딩하지 말 것.

## 7. 기술 스택
- 순수 HTML/CSS/JS로 단일 페이지 (빌드 도구 불필요)
- 스크롤 애니메이션은 CSS `@keyframes`의 `translateY` 또는 `requestAnimationFrame` 중 편한 방식 사용
- 모바일 Chrome / 글래스 내장 브라우저에서 동작 확인 필요
  (Web Bluetooth는 secure context 필수 → HTTPS로 서빙)

## 8. 결과물
- `index.html` (필요하면 css/js 분리)
- 간단한 `README.md`
  - Web Bluetooth 실험적 플래그 켜는 법
  - 테스트 모드(키보드 1~8) 사용법
  - 로컬에서 HTTPS로 띄우는 방법 (예: ngrok 등)
