# 메타 레이밴 디스플레이 도슨트 테스트 앱

박물관 도슨트 시스템 프로토타입의 1차 테스트용 웹앱입니다.
BLE 비콘(이름 `01`~`08`) 근처에 가면 해당 작품 안내 텍스트가 텔레프롬프터 방식으로
자동 스크롤되고, TTS로 읽어줍니다.

- 단일 파일 `index.html` (빌드 도구 불필요)
- 600x600 정사각형 뷰포트 기준 디자인, 검정 배경 / 흰 텍스트
- 가장자리 안전영역 확보, 가로 스크롤 없음

## ⚠️ 중요: 글래스는 BLE 스캔 불가 → 폰 리모컨 + 글래스 디스플레이 구조
메타 레이밴 디스플레이 글래스는 일반 브라우저가 아니라 제약된 **Web App 환경**(600×600,
D-pad 입력, 다크 배경 필수)이라 **블루투스 접근이 불가능**합니다. `requestLEScan`은
`chrome://flags` 실험 플래그가 필요한데 글래스에선 켤 수 없습니다.

대신 글래스는 **REST/WebSocket 네트워킹은 지원**하므로 다음 2-기기 구조로 동작합니다:

```
[폰] 비콘 선택(현재는 수동 버튼)  ──ntfy.sh 채널──▶  [글래스] 해당 도슨트 텍스트 표시
```

- 실시간 채널은 무설정 공개 pub/sub **ntfy.sh** 사용 (가입·백엔드 불필요)
- 폰과 글래스가 **같은 채널(토픽)** 을 써야 합니다. 기본값은 코드에 내장되어 있고
  `?ch=원하는토픽` 으로 바꿀 수 있습니다. (예: `...github.io/metadisplaytest/?ch=mychannel`)
- 공개 서비스이므로 토픽은 추측하기 어렵게(개인 식별 문자열) 두는 것을 권장합니다.

## 메인 화면 (모드 4개)

### ① 글래스 디스플레이 모드 (글래스에서 선택)
1. **연결 시작** 버튼(D-pad Enter) → ntfy 채널 구독
2. 폰에서 신호가 오면 해당 비콘의 도슨트 텍스트를 텔레프롬프터로 표시 (TTS는 베스트에포트)
3. 늦게 접속해도 최근 10분 내 마지막 신호를 받아 바로 표시, 연결 끊기면 자동 재연결

### ② 폰 리모컨 모드 (폰에서 선택)
1. `01`~`08` 버튼 그리드 → 탭하면 해당 비콘을 글래스로 전송
2. "대기(지우기) 전송" → 글래스 화면을 대기 상태로 되돌림
3. 글래스 없이도 무엇을 보냈는지 화면에서 즉시 확인 가능

### ③ 단독 도슨트 테스트 (BLE/시뮬레이터, PC·폰 단독)
1. 버튼 클릭 → 스캔 시작
2. 이름 `01`~`08` 디바이스 중 RSSI가 가장 강한 것을 현재 위치로 판단
   (노이즈 방지를 위해 약 2.5초간 최강 신호 유지 시 확정)
3. 위치가 바뀌면 화면 내용 교체 + 스크롤 처음부터 재시작 + TTS
4. 실제 비콘이 없으면 우측 상단 디버그 패널의 시뮬레이터/키보드 `1`~`8`로 테스트

### ④ 주변 BLE 스캔
- 주변의 모든 BLE 광고 패킷을 스캔해 **신호 세기 순으로 목록 표시**
- 이름 또는 제조사가 식별되는 실제 기기만 표시 (맥북·워치 등)
  - 이름이 없으면 제조사로 라벨링 (예: `Apple 기기`, `Samsung 기기`)
  - 이름·제조사 둘 다 없는 익명 기기는 기본 숨김 → 상단 체크박스로 전체 보기
- 각 기기마다 근접도 막대 + 실제 `dBm` 값 표시, **← 뒤로**로 메인 복귀

> 참고: Web Bluetooth는 프라이버시상 광고에 기기 이름을 잘 싣지 않습니다.
> 그래서 애플/삼성 기기 등은 사람이 정한 이름 대신 **제조사 라벨**로 뜨는 경우가 많습니다.

## Web Bluetooth 실험적 플래그 켜는 법
이 앱은 지속 스캔을 위해 실험적 API `navigator.bluetooth.requestLEScan()`을 사용합니다.
Chrome 계열 브라우저에서만 동작하며, 아래 플래그를 켜야 합니다.

1. 주소창에 `chrome://flags` 입력
2. **Experimental Web Platform features** 검색 → **Enabled**
3. 브라우저 재시작
4. (Android의 경우) 위치 권한 / 블루투스 권한 허용

> Web Bluetooth는 secure context가 필수이므로 반드시 **HTTPS**(또는 `localhost`)로 서빙해야 합니다.

## 테스트 / 디버그 모드
실제 비콘 8개가 없어도 테스트할 수 있습니다.

- 우측 상단 **디버그 ▾** 버튼으로 패널을 엽니다. (BLE 미지원 환경에선 스캔 시작 시 자동으로 열림)
- **신호 / 근접도**: 비콘 8개의 신호 세기를 "가까운 정도" 막대로 표시
  - `●` 노란색 = 현재 표시 중(확정), `◌` 파란색 = 곧 확정될 후보
  - 막대가 길수록(RSSI가 높을수록) 가까움. 옆에 실제 dBm 값 표시
- **근접 시뮬레이터**: 실제 비콘이 없을 때 슬라이더로 각 비콘의 신호를 흉내냄
  - 슬라이더를 오른쪽으로 올릴수록 그 비콘이 가까워짐 (맨 왼쪽 = 신호 없음)
  - 가장 강한 비콘이 약 2.5초 유지되면 실제 BLE와 동일하게 "현재 위치"로 확정
- **키보드 `1`~`8`**: 해당 번호 비콘을 최강으로 만들어 즉시 진입 (빠른 점프)
- **TTS 켜기/끄기** 토글
  - TTS On: 한국어(ko-KR) 음성으로 헤더+본문을 읽음
  - TTS Off: 비콘 진입 시 짧은 비프음 1회 재생

## HTTPS로 서빙하기

> Web Bluetooth는 secure context가 필수입니다. 글래스는 `localhost`가 아니라
> 네트워크 IP로 접속하므로 **반드시 HTTPS URL이 필요**합니다.
> (PC 크롬에서 키보드 테스트만 할 때는 `localhost`도 secure context로 인정됩니다.)

### 방법 A — GitHub Pages (배포됨, 추천)
이 저장소는 GitHub Pages로 호스팅됩니다. github.io는 자동으로 HTTPS를 제공하므로
글래스에서 바로 접속할 수 있습니다.

```
https://limjongyoon.github.io/metadisplaytest/
```

처음 배포하거나 다른 계정에 올릴 때:
```bash
git init -b main
git add .
git commit -m "도슨트 테스트 앱"
gh repo create metadisplaytest --public --source=. --remote=origin --push
# Pages 활성화 (main 브랜치 루트)
gh api -X POST repos/<USER>/metadisplaytest/pages \
  -f "source[branch]=main" -f "source[path]=/"
# 발급된 URL 확인 (빌드까지 1~2분)
gh api repos/<USER>/metadisplaytest/pages --jq .html_url
```
이후 코드를 바꾸면 `git push`만 하면 자동 재배포됩니다.

### 방법 B — ngrok 터널 (PC 빠른 테스트용 대안)
배포 없이 PC에서 임시 HTTPS URL을 띄울 때:
```bash
python3 -m http.server 8000          # 정적 서버
ngrok http 8000                      # 다른 터미널에서 HTTPS 터널
```
출력된 `https://xxxx.ngrok-free.app`을 글래스 브라우저에서 엽니다. (PC가 켜져 있어야 함)

## 데이터 / 커스터마이징
- 비콘 목록은 `index.html` 상단 `beacons` 배열에서 자동 생성됩니다(`01`~`08`).
- 본문은 `bodyLine`을 `BODY_REPEAT`(기본 50)회 반복해 긴 스크롤 콘텐츠를 자동 생성합니다.
- 스크롤 속도: `SCROLL_SPEED`(px/초), 신호 확정 시간: `CONFIRM_MS`,
  미감지 제외 시간: `STALE_MS` 상수로 조정합니다.
